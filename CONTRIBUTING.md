## How to contribute

1. Open an issue for bugs or feature ideas if you’d like to discuss first.
2. Fork the repo, create a branch, and make your changes.
3. Run tests (`make test`, and optionally `make test-integration`).
4. Open a pull request with a clear description of the change.

Please keep PRs focused and ensure the OpenAPI spec and docs stay in sync when you add or change API routes.
